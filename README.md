# Интерактивный калькулятор на JavaScript

## 📋 Описание проекта

Веб-приложение "Калькулятор" — это интерактивное приложение, разработанное с использованием HTML, CSS и современного JavaScript (ES6+). Приложение позволяет выполнять базовые арифметические вычисления с удобным пользовательским интерфейсом.

Проект разработан в рамках лабораторной работы №22 по изучению интерактивных веб-приложений на JavaScript.

## ⚙️ Функциональность

### Поддерживаемые операции

- **Сложение** (`+`)
- **Вычитание** (`-`)
- **Умножение** (`*`)
- **Деление** (`/`)
- **Работа с десятичными дробями** (`.`)
- **Очистка выражения** (`C`)
- **Вычисление результата** (`=`)

### Ограничения

- Ввод начинается только с цифры или знака минуса (нельзя начать с оператора `+`, `*`, `/`)
- Запрещён ввод двух операторов подряд (например, `5++3`)
- Нельзя ввести несколько точек в одном числе (например, `5.2.1`)
- Выражение не может заканчиваться оператором
- При делении на ноль выводится сообщение об ошибке

### Обработка ошибок

- **Некорректное выражение**: при попытке вычислить невалидное выражение на дисплее отображается `"Ошибка"`
- **Деление на ноль**: результатом будет `"Ошибка"`
- **Синтаксические ошибки**: перехватываются через `try...catch`
- После ошибки калькулятор переходит в специальное состояние — следующее нажатие любой кнопки (кроме `C`) автоматически очистит дисплей

## 🔧 Логика работы

### Последовательность действий при нажатии кнопок

1. **Пользователь нажимает кнопку** → срабатывает событие `click`
2. **`handleButtonClick(value)`** получает текст кнопки
3. **Проверка состояния:**
   - Если был флаг ошибки и нажата не `C` → автоматическая очистка
4. **Распределение действия:**
   - `C` → вызов `clear()`
   - `=` → вызов `calculate()`
   - Иначе → вызов `addToExpression(value)`

### Как работает валидация ввода

**Метод `addToExpression(value)` проверяет:**

1. **Начало выражения:**
   - Не может начинаться с оператора (кроме `-` для отрицательных чисел)

2. **Двойные операторы:**
   - Если последний символ — оператор, а вводится новый оператор:
     - Последний оператор заменяется на новый
     - Пример: `5+` → нажимаем `-` → `5-`

3. **Несколько точек:**
   - Разбивает выражение по операторам
   - Проверяет текущее число на наличие точки
   - Если точка уже есть — ввод блокируется

4. **Корректность символов:**
   - Разрешены только: `0-9`, `+`, `-`, `*`, `/`, `.`, `(`, `)`

### Как происходит вычисление

1. **Пользователь нажимает `=`** → вызывается `calculate()`

2. **Проверка валидности:**
   - Выражение не пустое
   - Не заканчивается оператором
   - Проходит проверку `isValidExpression()`

3. **Безопасное вычисление:**
   - `safeEvaluate(expr)` проверяет строку на допустимые символы
   - Используется `eval()` только после всех проверок

4. **Обработка результата:**
   - Если результат конечный (`isFinite`) → выводится на дисплей
   - Если бесконечность или NaN → ошибка
   - Результат записывается в `expression` (можно продолжать вычисления)

5. **Обработка ошибок:**
   - Любое исключение перехватывается `catch`
   - Устанавливается флаг `hasError = true`
   - На дисплее показывается `"Ошибка"`

# Крестики-нолики (Tic-Tac-Toe)

Интерактивное веб-приложение, разработанное в рамках лабораторной работы №23 по курсу Fullstack Development. Игра реализована на чистом JavaScript (Vanilla JS) с использованием HTML5 и CSS3.

## 1. Описание игры

### Правила игры
Классическая игра для двух игроков на поле 3×3.
- Игроки ходят по очереди, размещая свои символы (**X** или **O**) в свободных клетках.
- Первый ход всегда принадлежит игроку **X**.
- Цель игры — собрать линию из трёх своих символов подряд по горизонтали, вертикали или диагонали.
- Если все клетки заполнены, но линия не собрана, объявляется ничья.

### Логика победы/поражения
- **Победа**: Присуждается игроку, который первым замкнёт одну из 8 выигрышных комбинаций.
- **Поражение**: Наступает автоматически для проигравшего, когда соперник собирает линию.
- **Ничья**: Фиксируется, если на поле не осталось пустых клеток, а победитель не выявлен.

### Особенности реализации
- Использование **CSS Grid** для построения сетки игрового поля.
- Манипуляции с **DOM** для динамического обновления интерфейса без перезагрузки страницы.
- Хранение состояния игры в памяти браузера (переменные JavaScript).
- Блокировка ввода после завершения игры до момента перезапуска.

---

## 2. Игровая механика

### Как определяется победитель?
После каждого хода система проверяет массив состояния игры (`board`) на соответствие одному из 8 заранее заданных шаблонов победы (3 горизонтали, 3 вертикали, 2 диагонали). Если три клетки в любом шаблоне содержат одинаковый непустой символ текущего игрока, игра останавливается и объявляется победа.

### Как обрабатываются клики?
1. На каждую клетку повешен слушатель событий (`addEventListener('click')`).
2. При клике функция `handleCellClick`:
   - Считывает индекс клетки из атрибута `data-index`.
   - Проверяет, пуста ли клетка и активна ли игра.
   - Записывает символ текущего игрока в массив данных и отображает его в HTML.
   - Запускает функцию проверки результата (`checkResult`).

### Как работает перезапуск?
При нажатии на кнопку «Новая игра» вызывается функция `restartGame`, которая:
- Сбрасывает переменную текущего игрока на «X».
- Очищает массив `board`, заполняя его пустыми строками.
- Устанавливает флаг `gameActive` в `true`.
- Очищает текстовое содержимое всех визуальных клеток.
- Обновляет статусное сообщение.

---

## 3. Структура данных

### Массив `board`
Основная структура данных, представляющая игровое поле:
```javascript
let board = ["", "", "", "", "", "", "", "", ""];
```
- Это массив из 9 элементов.
- Индексы 0-2 соответствуют верхней строке, 3-5 — средней, 6-8 — нижней.
- Пустая строка "" означает свободную клетку.
- Значения "X" или "O" означают занятую клетку.

## Условия победы
Выигрышные комбинации хранятся в константе winningConditions — массиве массивов:
```javascript
const winningConditions = [
    [0, 1, 2], [3, 4, 5], [6, 7, 8], // Горизонтали
    [0, 3, 6], [1, 4, 7], [2, 5, 8], // Вертикали
    [0, 4, 8], [2, 4, 6]             // Диагонали
];
```
Каждый вложенный массив содержит индексы клеток, образующих линию.

### Хранение состояния игры

Состояние поддерживается тремя глобальными переменными:

1. board (Array): Текущая расстановка символов.
2. currentPlayer (String): Строка "X" или "O", указывающая, чей сейчас ход.
3. gameActive (Boolean): Флаг, разрешающий или запрещающий дальнейшие ходы.
---
## 4. Алгоритмы
#### Алгоритм проверки победы
 Реализован в функции checkResult():
1. Инициализируется флаг roundWon = false.
2. Запускается цикл по всем элементам winningConditions.
3. Для каждой комбинации извлекаются значения трех клеток из массива board.
4. Если хотя бы одна клетка пуста — проверка продолжается.
5. Если все три клетки равны между собой — флаг roundWon становится true, цикл прерывается.
6. Если roundWon === true: выводится сообщение о победе, игра останавливается (gameActive = false).
#### Алгоритм определения ничьи
Выполняется сразу после проверки победы (если победа не найдена):
1. Проверяется наличие пустых строк в массиве board через метод .includes("").
2. Если пустых строк нет (возвращает false), значит, поле заполнено полностью.
3. Выводится сообщение «Ничья!», игра останавливается.
#### Алгоритм смены игрока
Выполняется, если победа и ничья не зафиксированы:
1. Используется тернарный оператор для переключения значения:
```javascript
currentPlayer = currentPlayer === "X" ? "O" : "X";
```
2. Обновляется текстовое поле статуса информацией о ходе следующего игрока.
